# 🔒 Security Policy

## 🛡 Supported Versions

The following versions of GrubedX are currently supported with security updates:

| Version | Supported |
| ------- | --------- |
| 5.1.x   | ✅         |
| 5.0.x   | ❌         |
| 4.0.x   | ✅         |
| < 4.0   | ❌         |

> ⚠️ Only the latest stable and actively maintained versions receive security updates. Older versions may contain known vulnerabilities.

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### 📩 How to Report

* Open a **private security advisory** on GitHub (preferred), or
* Contact the maintainer directly (if applicable)

> ❗ Please **do not** open a public issue for security vulnerabilities.

---

## ⏱ Response Expectations

* **Initial response:** within 48–72 hours
* **Status update:** within 5–7 days
* **Resolution time:** depends on severity and complexity

---

## 🔍 What to Include

To help resolve the issue quickly, please include:

* A clear description of the vulnerability
* Steps to reproduce the issue
* Potential impact and affected areas
* Any suggested fixes (if available)

---

## ✅ Disclosure Process

* Vulnerabilities will be reviewed and validated
* If accepted:

  * A fix will be developed and released
  * A security update will be published
* If declined:

  * You will receive an explanation

---

## 🤝 Responsible Disclosure

We ask that you:

* Avoid publicly disclosing the issue until it has been resolved
* Give reasonable time for a fix to be implemented

---

## 🙏 Acknowledgements

We appreciate responsible disclosure and will credit contributors who help improve the security of this project (if desired).
